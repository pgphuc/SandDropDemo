PNG → Pixel Tool — Hướng dẫn sử dụng Convert ảnh PNG sang Pixel Data & chỉnh sửa ảnh pixel trong Level Editor Window

Tool này làm gì? Nhận 1 ảnh PNG → thu nhỏ (downscale) xuống lưới pixel nhỏ (điền rộng/cao riêng) → tự map mỗi ô về 1 màu trong danh sách GameObjectColor (gần nhất theo màu) → cho bạn sửa tay từng ô (vẽ / tẩy / undo) → lưu thẳng vào LevelData. Đây là tool dành cho Editor (không vào build).

1. Mở tool ở đâu

Tool không có cửa sổ riêng — nó là 1 panel nằm trong Level Editor Window.

  1. Mở cửa sổ: Ctrl + 8  (hoặc menu J Tools ▸ Level ▸ Level Editor).
  2. Chọn Level Database.
  3. Chọn 1 LevelData ở pane bên trái.
  4. Pane phải hiện 2 tab → bấm tab “Convert (PNG→Pixel)”.

2. Quy trình tổng quát

  1. Chọn ảnh PNG nguồn.
  2. Điền Size W×H (số ô ngang × dọc), chọn mode downscale và match (cách map màu).
  3. Bấm [Generate] → tool tự thu nhỏ ảnh và tô màu từng ô.
  4. Chỉnh tay nếu cần: chọn màu ở palette trái, vẽ trực tiếp lên canvas (Ctrl+Z để undo).
  5. Bấm [Confirm → Edit Level] để lưu & xem output.
Lưu ý quan trọng: preview overlay và undo stack chỉ nằm trong RAM — khi Unity recompile (domain reload) sẽ mất. Dữ liệu pixel đã Generate / Confirm thì được lưu vào asset, nhưng nếu mất preview cứ bấm Generate lại.

3. Tab Convert — thanh điều khiển

ControlÝ nghĩa & cách dùng
PNG Popup liệt kê mọi ảnh trong folder Assets/!_Game_SDK/Pictures. Nút ↻ = quét lại folder. Nhận mọi định dạng Unity import thành Texture2D (png/jpg/tga/psd…), không kén đuôi.
Size W×H 2 ô số: chiều rộng (W) × cao (H) tính theo SỐ Ô pixel. Ép đúng W×H (không giữ tỉ lệ ảnh gốc). Giá trị được nhớ qua phiên. Mặc định 32×32 khi chưa từng điền.
mode Cách gộp pixel khi thu nhỏ (xem mục 5): Block Average (mượt, nhiều sắc độ) hoặc Dominant (sắc cạnh, sạch viền). Có dòng mô tả ngay dưới popup.
match Cách map mỗi ô về GameObjectColor (xem mục 4): RgbEuclid hoặc Hue (mặc định).
[Generate] Chạy toàn bộ pipeline: downscale → map màu → ghi vào PixelLevelData → lưu asset.

4. Màu & cách map màu

Palette = toàn bộ giá trị enum GameObjectColor (bỏ None và Mix). Màu hiển thị mỗi ô do hệ thống tự cấp; màu nào chưa cấu hình thì dùng màu fallback dựng sẵn (màu magenta = báo màu đó chưa được cấu hình).

Hai chế độ map màu (match)

ModeHành viDùng khi
RgbEuclid Khớp theo RGB kể cả độ sáng → sát ảnh gốc. 2 sắc độ cùng tông (xanh sáng vs xanh tối) có thể ra 2 màu khác nhau. Muốn bám sát ảnh, ảnh đã ít màu & phẳng.
Hue mặc định Khớp theo TÔNG (HSV) → các sắc độ cùng tông gộp về 1 màu (xanh sáng + xanh tối → Blue). Đen → Black, trắng → White ổn định. Muốn output gọn về đúng các màu game, ít bị tách sắc độ.
Mẹo: nếu output bị “gộp quá tay” (mất chi tiết) → chuyển sang RgbEuclid. Nếu bị tách thành quá nhiều sắc độ lặt vặt → dùng Hue.

5. Kích cỡ & mode downscale

Tool ép đúng W×H ô bạn nhập (không giữ tỉ lệ). Mỗi ô gộp 1 block pixel nguồn theo mode:

ModeHành vi
Block Average
mặc định		 Trung bình RGB có trọng số alpha; chỉ gộp pixel “đặc” (alpha cao), bỏ fringe bán trong suốt ở rìa → hết viền nâu. Mượt, giữ nhiều sắc độ. Ảnh không có alpha (.jpg) → hành vi như trung bình thường.
Dominant Color Lấy màu xuất hiện nhiều nhất trong block (không trộn) → sắc cạnh, sạch viền, hợp art phẳng ít màu.

“Viền nâu” chỉ xuất hiện khi trộn fringe anti-alias của PNG có alpha. Dùng Dominant hoặc Block Average (bản mới đã bỏ fringe) để tránh.

6. Chỉnh sửa ảnh pixel (paint / eraser)

Sau khi Generate, dùng thanh Edit bar và palette trái để sửa tay:

Control / thao tácTác dụng
zoom (slider 4..48)Phóng to/thu nhỏ px mỗi ô. Hoặc Ctrl + cuộn chuột trên canvas.
brush (slider 1..16)Kích cỡ cọ (số ô mỗi cạnh). Tô 1 vùng vuông căn giữa con trỏ. 1 = 1 ô. Gõ số trực tiếp được.
Palette (cột trái)Click 1 hàng để chọn màu vẽ. Hàng Eraser (-1) = tẩy về ô trống.
Chuột trái click/kéoVẽ màu đang chọn (theo brush size).
Ctrl + ZUndo 1 nét vẽ (1 stroke = 1 snapshot).
Overlay preview gốc (toggle)Vẽ ảnh gốc mờ đè lên canvas để so khớp khi chỉnh.
Cách đọc canvas: ô trống = ô caro xám tối (để không lẫn với pixel màu trắng); ô có giá trị = màu GameObjectColor tương ứng (magenta = màu chưa có material). Lưới (grid lines) hiện khi zoom ≥ 6.

7. Đếm pixel mỗi màu

Panel “Pixel mỗi màu” (cột trái, có ở cả 2 tab) đếm số ô theo từng màu trong grid hiện tại: hàng Trống (-1) trước, rồi từng GameObjectColor có mặt (swatch + tên + số lượng). Giá trị “lạ” không còn trong palette hiển thị dạng #<n> (?).

8. Confirm → Edit Level

Bấm [Confirm → Edit Level] ở cuối tab Convert để lưu (SaveAssets) và chuyển sang tab “Edit Level”. Tab này hiện read-only: hiển thị “Output: W × H”, zoom, bảng đếm màu và canvas xem lại (không vẽ được). Đây là chỗ để mở rộng công cụ edit level về sau.

9. Lưu ý & lỗi thường gặp

  • Folder ảnh: đặt PNG vào Assets/!_Game_SDK/Pictures. Folder trống → dropdown PNG báo trống.
  • Ảnh lần đầu không convert được? Tool tự bật Read/Write cho texture (cần để đọc pixel) rồi reimport & load lại — bình thường. Nếu vẫn lỗi, kiểm tra texture có hợp lệ không.
  • Preview / undo mất sau recompile: chúng ở RAM. Cứ Generate lại; dữ liệu đã Confirm vẫn còn trong asset.
  • Màu ra magenta: màu đó chưa được cấu hình trong hệ thống màu → vẫn vẽ/lưu được bình thường, chỉ là swatch tạm.
  • Thêm màu mới vào palette: thêm member GameObjectColor với int mới (không chèn giữa, vì int đã được serialize vào data) + case màu fallback tương ứng.
Không chỉnh tay file .asset / .meta / .prefab / .unity. Mọi thay đổi màu/material chỉ làm trong Unity Inspector. Sau khi sửa code drawer/enum, verify bằng cách compile trong Unity Editor.
